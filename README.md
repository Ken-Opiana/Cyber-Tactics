## 🎮 About

**Cyber Tactics** is a 3D turn-based roguelite card battler built in [Godot](https://godotengine.org/). Each run takes you through a branching map of battles and story events where you assemble a deck, manage your resources turn by turn, and outwit enemies in tactical combat.

What sets it apart is its purpose: every encounter is designed to teach a **core cybersecurity concept**. The game blends narrative, strategy, and education so that learning security fundamentals feels like playing a game — not reading a textbook.

> Build a deck. Survive the run. Learn to defend.

---

## 🔐 Learn Cybersecurity Through Play

Cyber Tactics weaves real security ideas directly into its mechanics and story. Rather than memorizing definitions, you apply concepts under pressure as part of tactical decision-making, and an in-game **Codex** lets you revisit what you've encountered.

The goal is to make the fundamentals of cybersecurity approachable and memorable for learners, students, and anyone curious about how digital defense works — all through engaging, hands-on gameplay.

---

## ✨ Features

- 🃏 **Deck-building card combat** — Play cards each turn, target enemies, and chain tactics together to win battles.
- 🗺️ **Roguelite run structure** — Navigate a branching map of battle rooms and event rooms; every run is shaped by chance.
- 🎲 **Procedural variety** — A seeded random system keeps encounters fresh from one playthrough to the next.
- 🧊 **Retro-styled 3D visuals** — Low-resolution rendering with crisp, integer-scaled pixel presentation for a distinctive aesthetic.
- 💬 **Story-driven campaign** — A full dialogue and cutscene system delivers narrative between battles, organized into chapters.
- 📖 **In-game Codex** — Track and review the cybersecurity concepts you've learned along the way.
- ⚡ **Status effects & tactics** — Buffs, debuffs, and special effects add depth to every decision.
- 🎵 **Original presentation** — Custom UI theme, music, sound effects, screen-shake juice, and a custom cursor.

## 🕹️ How to Play

Cyber Tactics is played primarily with the **mouse**, with a few keyboard shortcuts. Exact card and ability effects are shown in-game.

| Input | Action |
|---|---|
| 🖱️ **Left Click** | Select / play cards, confirm, and interact |
| 🖱️ **Right Click** | Secondary action (cancel / inspect) |
| 🖱️ **Scroll Wheel** | Scroll and navigate |
| ⌨️ **W / ↑** | Navigate up |
| ⌨️ **S / ↓** | Navigate down |
| ⌨️ **Esc** | Pause |

## 🚀 Getting Started (Development)

Want to run the project from source or contribute? Here's how to get set up.

### Prerequisites

- **[Godot Engine 4.6](https://godotengine.org/download)** (standard / GDScript build — not the .NET/Mono version)
- **[Git](https://git-scm.com/)**

### Setup

1. **Clone the repository**
   ```bash
   git clone https://github.com/Ken-Opiana/Cyber-Tactics.git
   cd Cyber-Tactics
   ```

2. **Open the project in Godot**
   - Launch Godot 4.6.
   - Click **Import**, browse to the cloned folder, and select the **`project.godot`** file.
   - Open the project.

3. **Run the game**
   - Press **F5** (or the ▶ Play button) in the Godot editor.

> 💡 A debug **`cheat`** input (the **Q** key) is mapped in the project for development and testing.

## 📂 Project Structure

A high-level map of the repository to help you find your way around:

```
Cyber-Tactics/
├── art/                    # Sprites, textures, and visual assets
├── battles/
│   └── chapter1/           # Battle encounters for Chapter 1
├── characters/             # Playable character definitions and scenes
├── custom_resources/       # Custom Resource types (cards, stats, etc.)
├── dialogues/              # Dialogue content for story scenes
├── effect/                 # Visual and gameplay effects
├── enemies_stats/          # Enemy stat resources
├── event_rooms/
│   └── chapter_01/         # Non-combat story / event encounters
├── global/                 # Autoload singletons (see below)
├── scenes/                 # Core game scenes (menus, map, battle, etc.)
├── script_templates/       # Editor script templates
├── shared_cards/           # Cards available across characters
├── statuses/               # Status effects (buffs / debuffs)
├── test_data/              # Test and sample data
├── threads/                # Threading / background-loading helpers
├── cutscene.gd / .tscn     # Cutscene controller and scene
├── main_theme.tres         # Global UI theme
├── default_bus_layout.tres # Audio bus layout
└── project.godot           # Godot project configuration
```

### Core Systems (Autoloads)

The game is built around a set of global singletons defined in `global/`:

| Autoload | Responsibility |
|---|---|
| `Events` | Global signal/event bus for decoupled communication |
| `RNG` | Seeded randomness for roguelite runs |
| `GlobalTurnNumber` | Tracks the current turn in battle |
| `CardLibrary` | Central registry of card definitions |
| `CodexManager` | Manages the cybersecurity learning Codex |
| `DialogueManager` / `DialogueState` | Drives branching dialogue and story state |
| `PortraitRegistry` | Character portrait lookup |
| `ModelRegistry` | 3D model lookup |
| `MusicPlayer` / `SFXPlayer` | Audio playback |
| `Shaker` | Screen-shake effects |
| `SettingsManager` | Persists player settings |
| `StaticTransition` | Scene transition effects |
